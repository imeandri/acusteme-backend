1.3.1 Geonames

Datatype: GeoNames

CA element code: geonames_uso

Quicktip: Linked data field (GeoNames database). Enter the name of a city, in any language (e.g., Riva presso Chieri, Sulaymaniyah or سلێمانی , Beijing or 北京市, Tehran or تهران, ) and choose from the suggestions in the database. For more precise searches the Geonames codes can be used: ADM1 (first-order administrative division (for Italy, Region - the ADM1 Italy search returns all Italian regions - for the United States: the States - the search ); ADM2 (second-order administrative division, e.g. ADM2 Turin the province of Turin), ADM3 Turin (the city of Turin). The coordinates point to the geographic center of the identified area. For historical places the suffix h can be used, e.g.: ADM2H, Carbonia-Iglesias (former Italian province); PCLH (historical political entity, e.g., PCLH, German Democratic Republic; The use of location codes allows the definition of multiple areas (of physical geography and informal territories, e.g., STNB, scientific research base (STNB McMurdo Station, Antarctica), TERR, territory (TERR Western Sahara); GYSR, geyser, GYSR Géiser del Tatio, Chile, SA. For other codes conultare the documentation.

1.7 Ritual event/occasion

Datatype: InformationService

CA element code: ritual_event_ME

Repeatability: Yes

Quicktip: Linked Data field: indicate, if relevant, one or more ritual events or occasions to which the ethnomusicological resource refers. Search by entering at least three letters to activate the search, e.g., 'car' for 'Ivrea Carnival'; 'Now' or 'Naw' (or whatever alternative spelling the name is recorded with on wikidata, e.g., Central Kurdish 'نه ورۆز') for 'Nowruz'.

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT DISTINCT
  ?item
  ?keyLabel
  ?keyDesc
  ?lang
  ?layer
  ?placeLabel
WHERE {
  # Gate di curatela (solo focus)
  ?item wdt:P5008 wd:Q120325256 .

  # Match IT/EN su label o altLabel
  {
    ?item (rdfs:label|skos:altLabel) ?lab .
    FILTER(LANG(?lab) = "it")
    FILTER(REGEX(STR(?lab), "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
    BIND("it" AS ?lang)
    BIND(?lab AS ?rawKeyLabel)
  }
  UNION
  {
    ?item (rdfs:label|skos:altLabel) ?lab .
    FILTER(LANG(?lab) = "en")
    FILTER(REGEX(STR(?lab), "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
    BIND("en" AS ?lang)
    BIND(?lab AS ?rawKeyLabel)
  }

  # Description (fallback incrociato IT-EN)
  OPTIONAL { ?item schema:description ?descIt FILTER(LANG(?descIt)="it") }
  OPTIONAL { ?item schema:description ?descEn FILTER(LANG(?descEn)="en") }
  BIND(
    IF(?lang="it", COALESCE(?descIt, ?descEn),
                 COALESCE(?descEn, ?descIt)
    ) AS ?rawKeyDesc
  )

  # Luogo opzionale (location, se c'è)
  OPTIONAL {
    ?item wdt:P276 ?place .
    OPTIONAL { ?place rdfs:label ?pIt FILTER(LANG(?pIt)="it") }
    OPTIONAL { ?place rdfs:label ?pEn FILTER(LANG(?pEn)="en") }
    BIND(STR(COALESCE(?pIt, ?pEn, "")) AS ?placeLabel)
  }

  # Layer fisso
  BIND("ritual_occasion" AS ?layer)

  # Stringhe plain per CA
  BIND(STR(?rawKeyLabel) AS ?keyLabel)
  BIND(STR(COALESCE(?rawKeyDesc, "")) AS ?keyDesc)
}
ORDER BY ?lang ?keyLabel
LIMIT 200
