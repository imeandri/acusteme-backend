Datatype: List

CA element code: linkscan_type_ME

Quicktip: Describes the technology or method used to acquire the 3D representation. Examples: "laser scanning," "photogrammetry," "structured light," "CT scan." Useful for assessing the accuracy and nature of the data acquired.

1.1.5 Resolution

Datatype: List

CA element code: link3D_resolution_ME

Quicktip: Indicates the level of detail of the 3D representation, expressed in points per unit (e.g., points/mm) or polygon density. Useful value for comparing accuracy between different acquisitions or for restoration, printing or analysis purposes.

1.1.6 Format

Datatype: List

CA element code: link3D_format_ME

Quicktip: Specifies the format of the 3D file. Common examples: .obj, .stl, .ply, .glb, .fbx, .3mf. Essential to ensure compatibility with visualization, modeling, or preservation software.

1.1.7 Scale

Datatype: List

CA element code: link3D_scale_ME

Quicktip: Indicates the ratio of the size of the 3D model to the actual object. Example: 1:1 for full-scale model, 1:10 for reduced model. Important for understanding the accuracy of the representation compared to the original.
