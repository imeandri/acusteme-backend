1.4.2 Intrumentation – summary

Datatype: Text

CA element code: organico_sintetico_elab

Quicktip: Summary description of the ensemble processed, according to SBN MARC conventions (U9B.2 - M8B2). Contain only the number of solo voices/instruments (followed by "V" or "str") and/or the name of the ensemble (e.g., "2str," "Choir(4V)"). Use coded abbreviations, separated by comma, without spaces, in accordance with the SBN Music guide (U9B.2 - M8B2)

1.4.3 Intrumentation – detailed (SBN MARC)

Datatype: Text

CA element code: organico_analitico_elab

Quicktip: Analytical detail of the processed staff, according to SBN MARC conventions (U9B.3 - M8B3). Includes each individual voice or instrument with coded abbreviations (e.g., "pf-solo,2fl,2ob"), separated by comma, no spaces. Refer to the SBN Music guide (U9B.3 - M8B3) for schema and allowed codes

1.5.4 suffix

Datatype: List

CA element code: suffix_ME

Quicktip: Optional field for adding qualifiers or suffixes that further specify the medium of performance. Suffixes serve to refine the description of the ensemble or performance, indicating particular characteristics of the performance such as: One-handed performances (one hand) Performances shared by several musicians on one instrument (two players on one instrument) Compositions for several hands (three hands, four hands, six hands, eight hands) Prepared instrument (prepared) Recorded (recorded) Amplified These suffixes are used to semantically enrich the "medium of performance" field, without altering its main structure, but adding specific performance information, which is essential for indexing and analytical cataloging of contemporary or experimental music.

1.9 Representation or recording

Datatype: Container

CA element code: representation_recording_CN

Quicktip: Structured section dedicated to the description of performances, public performances, or audio/video recordings related to the documented work or resource. Allows you to specify information related to the nature of the event (e.g., stage performance, live recording, remastering), its temporal, geographic, and institutional location. Includes data on genre of performance, date, period, location, occasion, and other relevant information. If relevant, cross-reference with other fields such as Date of performance, Location, Venue, Occasion, Type of event.

Usage examples: {to be completed}

1.9.1 Representation type

Datatype: List

CA element code: genre_of_representation_ME

Quicktip: Indicate the genre of representation of the resource, choosing from the default values. This field allows you to distinguish the nature of the event or sound/visual document associated with the work. Available values are: Representation: stage performance or general performance (theater, music, dance, etc.); First performance: first documented public performance of a work; Musical sound recording: audio recording in a studio or controlled context; Live recording: recording made during a public performance; Remastering: technical reworking of a pre-existing recording (e.g., audio enhancement); Unspecified: to be used only when the genre of performance cannot be determined.

Usage examples: {to be completed}

The system supports date-range and uncertain forms (e.g., 1821?, 085.-089.) according to SBN and ISBD rules
