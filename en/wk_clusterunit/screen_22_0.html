Datatype: Container

CA element code: ord_tit_CN

Quicktip: Section dedicated to the management of the work's sorting title, useful for alphabetical placement, indexing and normalization of titles, especially in the presence of initial articles or insignificant titles.

Usage examples: {to be completed}

1.4.1 Sorting Title

Datatype: Text

CA element code: ord_tit_ME

Quicktip: Enter the normalized title of the work according to SBN conventions for alphabetical sorting. The sorting title: excludes initial articles and non-significant expressions (e.g., "The," "The," "An") maintains the natural order of significant words does not contain punctuation marks that are not necessary for identification It is used for alphabetical sorting in catalogs and for generating uniform titles. 🔗 For complete criteria, refer to the official guide: Guide to Cataloging in SBN - Sorting

Usage examples: {to be completed}

1.4.2 Non-significant title

Datatype: List

CA element code: nonsignificanttitle_ME

Quicktip: Check to be enabled if the work does not have a meaningful title (e.g., "Concerto No. 2," "Sonata," "Etude"). In this case, the title has no independent identifying function and must be supplemented by other elements (author, numbering, tonality, etc.) to ensure uniqueness. This option guides the construction of a uniform or conventional title for cataloguing purposes.

1.5 Content language (ISO 639-3)

Datatype: InformationService

CA element code: cont_lang_ME

Repeatability: Yes

Quicktip: Linked Data field.Indicate the main language or languages ​​of the content.Start the Linked Research given by entering at least three letters using the ISO language code (e.g. 'kmr' for the northern Kurdish) or the name language ('Ital' for 'Italian' or 'language of Italian signs').Register all the languages ​​of the content, or record at least the main language and indicate that the resource is multilingual in the next field.Repeatable element.

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT DISTINCT ?item ?itemLabel ?itemDescription ?value ?lang
WHERE {
  {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "it") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
UNION
 {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "en") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  UNION
  {
    ?item wdt:P220 ?value .
  FILTER (REGEX(?value, "***PLACEHOLDER***", "i"))  
    }  
    SERVICE wikibase:label { bd:serviceParam wikibase:language "it, en". }
BIND(LANG(?itemLabel) AS ?lang)
 }
