1.1 Area of publication, production and distribution

Datatype: Text

CA element code: pubprod_area_ME

Quicktip: Record information pertinent to the publication according to bibliographic standards (e.g., 'Cavoretto : [autograph], 21/22 8bre 1900'). You enter place names or geographic areas as given in the resource or inferred from external sources or analysis of the resource itself. Transcribe the place in the language and form in which it appears, maintaining spelling and grammatical case.

1.2.2 Date code

Datatype: List

CA element code: cod_data

Quicktip: Depending on the date type code, the fields Date 1 and Date 2 have different meanings. An example of a common case for published resources is code D (Monograph in one or more units, published in a single certain or probable year). Date 1 contains the unique year of publication of the monographic resource in one or more units, while Date 2 is absent. For all other cases, please refer to the documentation for examples.

1.4 Places

Datatype: Container

CA element code: biblio_res_place_CN

Repeatability: Yes

Quicktip: Enter the name of the city, locality, or geographical area as indicated in the resource or deduced from external sources or from the examination of the resource itself. Transcribe the place in the language and form in which it appears, respecting its spelling and grammatical case. Include the prepositions preceding the name of the place, as well as any accompanying words or phrases. The 'Linked data Geonames' field allows for the identification of a place in a coded form, while in the subsequent field ('Indicazione di luogo - testo libero') the indication of the place should be transcribed as it appears

1.4.2 GeoName

Datatype: GeoNames

CA element code: geonames_pub_place

Quicktip: Linked data field (GeoNames database). Enter the name of a city in any language (e.g., Riva near Chieri, Sulaymaniyah or سلێمانی, Beijing or 北京市, Tehran or تهران) and choose from the database's suggestions. For more precise searches, Geonames codes can be used: ADM1 (first-order administrative division, for example, Region for Italy - searching "ADM1 Italy" returns all Italian regions - for the United States: the States - the search); ADM2 (second-order administrative division, e.g., ADM2 Turin for the province of Turin), ADM3 Turin (the city of Turin). The coordinates point to the geographical center of the identified area. For historical places, the suffix 'h' can be used, for example, ADM2H, Carbonia-Iglesias (former Italian province); PCLH (historical political entity, e.g., PCLH, German Democratic Republic; Using location codes allows the definition of multiple geographical areas and informal territories, for example, STNB, scientific research base (STNB McMurdo Station, Antarctica), TERR, territory (TERR Western Sahara); GYSR, geyser (GYSR El Tatio Geyser, Chile, SA). For other codes, please refer to the documentation.
