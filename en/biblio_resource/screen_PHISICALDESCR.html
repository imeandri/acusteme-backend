1.1 Material designation and extension (main)

Datatype: Container

CA element code: material_desc_extension1_CN

Quicktip: Use the coded fields to indicate the specific designation of the material, the extension, and, if relevant, the system or format used. Examples: 1: '3' (Number) 'videocassette' (Specific material designation) 'VHS' (Format) 'approximately 111 min' (Extension). 2: '1' (Number) 'score' (Specific material designation) 'CXLVI, 291 p.' (Extension). If the coded fields are not applicable, use the next free-text field 'Material designation and extension'.

1.2.1 Connector

Datatype: List

CA element code: connector

Quicktip: Use ',' to link resources of equal importance. For example, a resource consisting of 1 manuscript score (47 p.) and 1 audio cassette (15 min 4 s): in the first container, report the first resource; in the second, report the second resource, using the ',' connector. Use 'in' if the indicated number of units does not correspond to the number of physically separate components (excluding any attachments), for example, 1 score 'in' 2 volumes, 3 parts 'in' 12 files. Use '+' for attachments, e.g., 15 parts "+" 2 orchestra layout sheets.

1.4 Carrier/resource dimensions

Datatype: Container

CA element code: encumbrance_CN

Quicktip: In the 'Measurements' field, give measurements in centimeters, except for manuscripts, which should be expressed in millimeters (mm), with the height per base of a representative sheet. The standard dimensions of audio and video recordings can be omitted if they are implicit in the specific material designation. The indication of weight is optional and should only be provided for objects. If the dimensions are multiple, use the next field 'Dimensions' (free text) for example, for manuscript volumes with significantly different dimensions, or for folded original pamphlets or sheets, where both the open and folded dimensions are indicated.

1.5.1 Resource main materials

Datatype: InformationService

CA element code: main_material_ME

Quicktip: Linked data field. If the information is not implicit in the designation, the material can be indicated here (e.g., for a manuscript: parchment; rice paper; for other supports: paper, metal, plastic; for an object or realia: ivory, stone). For some supports, the material applied to the support can be optionally indicated (e.g., chromium dioxide for an audio cassette).

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT DISTINCT ?item ?itemLabel ?itemDescription ?lang 
WHERE {
  {
    ?item ?p ?value .
    VALUES ?p {wdt:P4801 wdt:P9112}
    ?item (rdfs:label|skos:altLabel) ?itemLabel.
    FILTER (REGEX(?value, "mmaterial|3dmat|graphicspsm", "i"))
    FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  UNION
  { 
    ?item wdt:P1014 ?value .
    VALUES ?item {wd:Q226697 wd:Q11472 wd:Q1143474}
    VALUES ?value {"300014161" "300014109" "300011851"}
    ?item (rdfs:label|skos:altLabel) ?itemLabel.
    FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  FILTER(LANGMATCHES(LANG(?itemLabel), "it") || LANGMATCHES(LANG(?itemLabel), "en"))
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), "it") || LANGMATCHES(LANG(?itemDescription), "en"))
    FILTER(LANG(?itemLabel) = LANG(?itemDescription))
  SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE], en, it". }
BIND(LANG(?itemLabel) AS ?lang)

} 
order by ?item
