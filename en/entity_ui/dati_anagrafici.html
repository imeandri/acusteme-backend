1.4.2 Correspondence with Wikidata entity

Datatype: InformationService

CA element code: wikidata_correlation_ME

Quicktip: Linked Data field for selecting the corresponding entity in Wikidata. Association with a Wikidata entity (e.g., Q5582) allows integration with a large semantic graph and inheritance of external properties (e.g., VIAF, ISNI, BNF, LC). It is recommended to always validate that the selected entity is correct and up-to-date.

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

select distinct ?item ?itemLabel ?itemDescription ?lang 
where {
   SERVICE wikibase:mwapi {
      bd:serviceParam wikibase:endpoint "www.wikidata.org";
                      wikibase:api "EntitySearch";
                      mwapi:search "***PLACEHOLDER***";
                      mwapi:language "it".
      ?item wikibase:apiOutputItem mwapi:item.
      ?num wikibase:apiOrdinal true.
  }
  SERVICE wikibase:label {
                   bd:serviceParam wikibase:language "[AUTO_LANGUAGE],it,en" .
                   }
  ?item (rdfs:label|skos:altLabel) ?itemLabel.
  OPTIONAL {
     ?item schema:description ?itemDescription.
      FILTER(LANGMATCHES(LANG(?itemDescription), "it") || LANGMATCHES(LANG(?itemDescription), "en"))
   }
  FILTER(LANGMATCHES(LANG(?itemLabel), "it") || LANGMATCHES(LANG(?itemLabel), "en"))
      FILTER(LANG(?itemLabel) = LANG(?itemDescription))
    BIND(LANG(?itemLabel) AS ?lang)
}

1.4.3 VIAF (from Wikidata)

Datatype: InformationService

CA element code: VIAF_IS

Quicktip: Linked Data derived field (read-only or suggested) that retrieves the Virtual International Authority File (VIAF) identifier corresponding to the entity, if present in Wikidata. VIAF links authority files from national and international libraries (LC, BNF, DNB, etc.) into a single shared entity.

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT distinct ?item ?itemLabel ?itemDescription ?value ?lang ?ViafLink
WHERE {
  SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE],it,en". }
  
  ?item wdt:P214 ?value . 
  SERVICE wikibase:mwapi {
      bd:serviceParam wikibase:endpoint "www.wikidata.org";
                      wikibase:api "EntitySearch";
                      mwapi:search "***PLACEHOLDER***";
                      mwapi:language "it".
      ?item wikibase:apiOutputItem mwapi:item.
      ?num wikibase:apiOrdinal true.
  }
  ?item (rdfs:label|skos:altLabel) ?itemLabel.
  OPTIONAL {
     ?item schema:description ?itemDescription.
      FILTER(LANGMATCHES(LANG(?itemDescription), "it") || LANGMATCHES(LANG(?itemDescription), "en"))
   }
  FILTER(LANGMATCHES(LANG(?itemLabel), "it") || LANGMATCHES(LANG(?itemLabel), "en"))
      FILTER(LANG(?itemLabel) = LANG(?itemDescription))
    BIND(LANG(?itemLabel) AS ?lang)
   BIND(
  CONCAT("VIAF: ", "https://viaf.org/viaf/", ?value) AS ?ViafLink
)
  
 }
order by ?item

1.5 ICCU qualification codes

Datatype: Container

CA element code: iccu_qual_codes_CN

Quicktip: Container for the assignment of ICCU codes for qualifying onomastic forms. These codes indicate the nature of the name form according to SBN regulations: E.g., "Accepted form," "Variant form." They are used in authority control contexts to specify the role of the string within the structure of the authority record. Refer to the official SBN documentation (ICCU) for coding: https://norme.iccu.sbn.it/index.php?title=Qualificazioni

1.5.1 Name type

Datatype: List

CA element code: type_of_name_ME

Quicktip: Controlled vocabulary field, conforming to ICCU/SBN specifications, indicating the structure and form of the agent's name (natural person). It is used to distinguish how authorized access is constructed in authority records, according to the direct or reverse form and the number of elements in the principal name. Available values: A - Person name in direct form, whose main group consists of only one element (e.g. Dante) B - Person name in direct form, whose main group consists of several elements (e.g. Giuseppe Verdi) C - Person name in reverse form, whose main group consists of only one element (e.g. Dante, poet) D - Person name in reverse form, whose main group consists of several elements (e.g. Verdi, Giuseppe) See the section on name qualifications in the ICCU Standards portal for more details.

1.5.2 Name form

Datatype: List

CA element code: form_of_name_ME

Quicktip: Linked Data field specifying the form taken by the name in the present occurrence. This can be: Accepted/authorized form Variant form (e.g., transliterations, alternative names, pseudonyms). The distinction is useful to indicate whether the name represents the preferred heading or an alternate form. Please refer to the official SBN (ICCU) documentation on onomastic forms for the codes and criteria to be used: https://norme.iccu.sbn.it/index.php?title=Qualificazioni
