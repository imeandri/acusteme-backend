1.3 Cue Entry

Datatype: Container

CA element code: silentfilm_CN

Quicktip: Each numbered line within a document defined as a Cue Sheet corresponds to the description of the musical element that should accompany the film image at a given moment. In the context of silent films, in addition to the opening of the suggested musical piece for soundtracking, other information may appear, such as the title, placement within the film or scene, dramatic function, authors of the arrangement, publisher, duration, etc.

1.3.1 Dramatic Category

Datatype: InformationService

CA element code: dramaticcategory_ME

Quicktip: Linked Data field. Controlled terms identifying the Dramatic Category of the music piece when expressed in the title or title elements related to the semantic-mood intent of the piece, as stated by the composer or by the publisher of film music collections. For example, "Misterioso-Pizzicato, Storm, Orientale..."

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT DISTINCT ?item ?itemLabel ?itemDescription
WHERE {
  # Selezioniamo gli elementi che sono istanze di Q126077457
  ?item wdt:P31 wd:Q126077457.
  
  # Controlliamo etichette e alias per la stringa "Cabiria"
  {
    ?item rdfs:label ?itemLabel.
    FILTER((LANG(?itemLabel)) = "en")
    FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  UNION
  {
    ?item skos:altLabel ?itemLabel.
    FILTER((LANG(?itemLabel)) = "en")
    FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }

  # Opzionalmente, otteniamo la descrizione se disponibile
  OPTIONAL {
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), "en"))
  }
}
ORDER BY ?item
LIMIT 100

1.3.3 Key (IFLA from Wikidata)

Datatype: InformationService

CA element code: cuetonalityinc_ME

Quicktip: Linked Data field. Selects the tonality of the musical work using standardized terms derived from the IFLA Thesaurus for tonalities (in URI format), with matches in Wikidata. This field ensures semantic alignment between music cataloging and international models (e.g., UNIMARC, RDA).

Linked Data field: for the allowed terms, updated on , click the following link to run the query in real time: Run Query (for complex queries, wait several seconds before results are returned).

SELECT distinct ?item ?itemLabel ?itemDescription ?value  
WHERE {
  {?item wdt:P9112 ?value;
        rdfs:label ?itemLabel.
   FILTER (REGEX(?value, "key#", "i"))
 FILTER((LANG(?itemLabel)) = "it") 
 FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
 OPTIONAL {
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), LANG(?itemLabel)))
  }
  }
UNION
 {
   ?item wdt:P9112 ?value;
     rdfs:label ?itemLabel.
   FILTER (REGEX(?value, "key#", "i"))
 FILTER((LANG(?itemLabel)) = "en") 
 FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
 OPTIONAL {
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), LANG(?itemLabel)))
  }
  }
        SERVICE wikibase:label { bd:serviceParam wikibase:language "it, en". }

 }

ORDER BY ?item
