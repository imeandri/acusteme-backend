1.3.1 Denominazione generale del processo (Linked data)

Datatype: InformationService

CA element code: process_denomination

Quicktip: Campo linked data. Termini controllati che identificano il tipo generale di processo musicale coinvolto, secondo vocabolari strutturati o linked data. Collega la pratica descritta a un concetto standardizzato per garantire interoperabilità semantica.

Campo Linked Data: Per i termini ammissibili, aggiornati al , clicca sul seguente link per eseguire la query in tempo reale: Esegui Query (attenzione: per query complesse occorre attendere diversi secondi prima della restituzione dei risultati).

select ?item ?itemLabel ?itemDescription ?lang
where {
   SERVICE wikibase:mwapi {
      bd:serviceParam wikibase:endpoint "www.wikidata.org";
                      wikibase:api "EntitySearch";
                      mwapi:search "***PLACEHOLDER***";
                      mwapi:language "it".
      ?item wikibase:apiOutputItem mwapi:item.
      ?num wikibase:apiOrdinal true.
  }
  SERVICE wikibase:label {
                   bd:serviceParam wikibase:language "it, en" .
                   }
 
    ?item (rdfs:label|skos:altLabel) ?itemLabel.
  
     OPTIONAL {?item schema:description ?itemDescription.
                         FILTER(LANGMATCHES(LANG(?itemDescription), "it") || LANGMATCHES(LANG(?itemDescription), "en"))}
    FILTER(LANGMATCHES(LANG(?itemLabel), "it") || LANGMATCHES(LANG(?itemLabel), "en"))
    FILTER(LANG(?itemLabel) = LANG(?itemDescription))
    BIND(LANG(?itemLabel) AS ?lang)


}
order by ?item

1.3.2 Date del processo

Datatype: DateRange

CA element code: processdates

Quicktip: Campo 'daterange', inserire date singole. Esempi: '30 Agosto 1746', '1900', '7 Giugno 2007', 'Giugno 2016'. Collective access supporta anche date-range ed espressioni di date incerte nei campi date-range, ma è consigliabile utilizzare le normative bibliografiche, che esprimono intervalli incerti attraverso il metodo DATA 1-DATA2 (v. documentazione).
