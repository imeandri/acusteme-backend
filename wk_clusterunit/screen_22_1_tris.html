1.3 Genere

Datatype: InformationService

CA element code: mus_gen_ME

Ripetibilità: Sì

Quicktip: Campo Linked Data. Indicare il genere musicale dell’opera (es. sinfonia, opera lirica, cantata, messa), secondo vocabolario controllato.

Campo Linked Data: Per i termini ammissibili, aggiornati al , clicca sul seguente link per eseguire la query in tempo reale: Esegui Query (attenzione: per query complesse occorre attendere diversi secondi prima della restituzione dei risultati).

SELECT DISTINCT ?item ?itemLabel ?itemDescription ?value ?lang
WHERE {
    ?item wdt:P4953 ?value.
    #FILTER (REGEX(?value, "gf", "i"))
        ?item (rdfs:label|skos:altLabel) ?itemLabel.
    FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
    ?item(rdfs:label|skos:altLabel) ?itemLabel.
     
    FILTER(LANGMATCHES(LANG(?itemLabel), "it") || LANGMATCHES(LANG(?itemLabel), "en"))
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), "it") || LANGMATCHES(LANG(?itemDescription), "en"))
    FILTER(LANG(?itemLabel) = LANG(?itemDescription))
   
  BIND(LANG(?itemLabel) AS ?lang)
  SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE], en, it". } 

} 
order by ?item

1.4 Forma musicale (IFLA)

Datatype: InformationService

CA element code: mus_form_ME

Ripetibilità: Sì (max 3 volte)

Quicktip: Campo Linked Data. Specificare la forma musicale secondo la terminologia codificata IFLA UNIMARC (es. tema con variazioni, fuga, rondò, suite, sonata-forma).

Campo Linked Data: Per i termini ammissibili, aggiornati al , clicca sul seguente link per eseguire la query in tempo reale: Esegui Query (attenzione: per query complesse occorre attendere diversi secondi prima della restituzione dei risultati).

SELECT distinct ?item ?itemLabel ?itemDescription ?value ?lang ?iflaLink
WHERE {
  {?item wdt:P9112 ?value;
        rdfs:label ?itemLabel.
   FILTER (REGEX(?value, "fom#", "i"))
 FILTER((LANG(?itemLabel)) = "it") 
 FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
 OPTIONAL {
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), LANG(?itemLabel)))
  }
  }
UNION
 {
   ?item wdt:P9112 ?value;
     rdfs:label ?itemLabel.
   FILTER (REGEX(?value, "fom#", "i"))
 FILTER((LANG(?itemLabel)) = "en") 
 FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
 OPTIONAL {
    ?item schema:description ?itemDescription.
    FILTER(LANGMATCHES(LANG(?itemDescription), LANG(?itemLabel)))
  }
  }
        SERVICE wikibase:label { bd:serviceParam wikibase:language "it, en". }
BIND(LANG(?itemLabel) AS ?lang)
  
    BIND(
  CONCAT("IFLA TERM: ", "https://www.iflastandards.info/unimarc/terms/", ?value) AS ?iflaLink
)

  
 }

ORDER BY ?item

1.7 Luogo di composizione

Datatype: GeoNames

CA element code: rapprrec_place_GEO

Quicktip: Campo Linked Data (Geonames) per indicare il luogo geografico in cui l’opera è stata registrata o rappresentata. Inserire il nome di una città, in qualsiasi lingua (Es. Riva presso Chieri, Sulaymaniyah o سلێمانی , Pechino o 北京市, Teheran o تهران, ) e scegliere tra le proposte della base di dati. Per ricerche più precise possono essere utilizzati i codici Geonames: ADM1 (first-order administrative division (per l'Italia, Regione – la ricerca ADM1 Italia restituisce tutte le regioni italiane – per gli Stati Uniti: gli Stati – la ricerca ); ADM2 (second-order administrative division, es ADM2 Torino la provincia di Torino), ADM3 Torino (la città di Torino). Le coordinate puntano al centro geografico dell'area individuata. Per i luoghi storici è possibile utilizzare il suffisso h, ad esempio: ADM2H, Carbonia-Iglesias (ex provincia italiana); PCLH (historical political entity, es. PCLH, Repubblica Democratica Tedesca; L'uso dei codici di localizzazione consente la definizione di molteplici aree (di geografia fisica e territori informali, ad esempio STNB, scientific research base (STNB Stazione McMurdo, Antartide), TERR, territory (TERR Sahara Occidentale); GYSR, geyser, GYSR Géiser del Tatio, Cile, SA. Per altri codici conultare la documentazione.

1.9 Data di composizione/realizzazione

Datatype: DateRange

CA element code: comp_date_ME

Quicktip: Indicare la data o l’intervallo temporale della composizione o della prima realizzazione. Esempi: 1876, tra 1782 e 1784, inizio XX sec.. Campo 'daterange', inserire date singole. Esempi: '30 Agosto 1746', '1900', '7 Giugno 2007', 'Giugno 2016'. Collective access supporta anche date-range ed espressioni di date incerte nei campi date-range, ma è consigliabile utilizzare le normative bibliografiche, che esprimono intervalli incerti attraverso il metodo DATA 1-DATA2 (v. documentazione).

I dati possono essere utili per l’indicizzazione, l’identificazione dell’opera, e il confronto tra versioni.

Esempi d'uso: {to be completed}

1.10.1 Organico sintetico

Datatype: Text

CA element code: organico_sintetico

Quicktip: Descrizione sintetica dell’organico elaborato, secondo le convenzioni SBN MARC (U9B.2 – M8B2). Contiene soltanto il numero di voci/strumenti solisti (seguite da "V" o "str") e/o il nome del complesso (es. "2str", "Coro(4V)"). Usare le abbreviazioni codificate, separate da virgola, senza spazi, in accordo con la guida SBN Musica (U9B.2 – M8B2)

Esempi d'uso: {to be completed}

1.10.2 Organico analitico (SBN MARC)

"pf-solo,2fl,2ob"), separate da virgola, senza spazi. Consultare la guida SBN Musica (U9B.3 – M8B3) per schema e codici ammessi

1.11.4 suffisso

Datatype: List

CA element code: suffix_ME

Quicktip: Campo opzionale per l’aggiunta di qualificatori o suffissi che specificano ulteriormente le modalità esecutive del medium of performance. I suffissi servono a raffinare la descrizione dell’organico o dell’esecuzione, indicando caratteristiche particolari della performance come: Esecuzioni a una sola mano (one hand) Esecuzioni condivise da più musicisti su un unico strumento (two players on one instrument) Composizioni per più mani (three hands, four hands, six hands, eight hands) Strumento preparato (prepared) Registrato (recorded) Amplificato (amplified) Questi suffissi sono utilizzati per arricchire semanticamente il campo “medium of performance”, senza alterarne la struttura principale, ma aggiungendo informazioni esecutive specifiche, fondamentali per l’indicizzazione e la catalogazione analitica della musica contemporanea o sperimentale.
