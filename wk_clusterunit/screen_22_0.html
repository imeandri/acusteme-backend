1.4 Ordinamento

Datatype: Container

CA element code: ord_tit_CN

Quicktip: Sezione dedicata alla gestione del titolo di ordinamento dell’opera, utile per la collocazione alfabetica, l’indicizzazione e la normalizzazione dei titoli, soprattutto in presenza di articoli iniziali o titoli non significativi.

Esempi d'uso: {to be completed}

1.4.1 Titolo d'ordinamento

Datatype: Text

CA element code: ord_tit_ME

Quicktip: Inserire il titolo normalizzato dell’opera secondo le convenzioni SBN per l’ordinamento alfabetico. Il titolo d’ordinamento: esclude gli articoli iniziali e le espressioni non significative (es. “Il”, “La”, “Un”) mantiene l’ordine naturale delle parole significative non contiene segni di punteggiatura non necessari all’identificazione È utilizzato per l’ordinamento alfabetico nei cataloghi e per la generazione dei titoli uniformi. 🔗 Per i criteri completi, fare riferimento alla guida ufficiale: Guida alla catalogazione in SBN – Ordinamento

Esempi d'uso: {to be completed}

1.4.2 Titolo non significativo

Datatype: List

CA element code: nonsignificanttitle_ME

Quicktip: Spunta da attivare se l’opera non presenta un titolo significativo (es. "Concerto n. 2", "Sonata", "Studio"). In tal caso, il titolo non ha funzione identificativa autonoma e deve essere integrato da altri elementi (autore, numerazione, tonalità ecc.) per garantirne l’univocità. Questa opzione guida la costruzione di un titolo uniforme o titolo convenzionale per scopi catalografici.

1.5 Lingua/e del contenuto (ISO 639-3)

Datatype: InformationService

CA element code: cont_lang_ME

Ripetibilità: Sì

Quicktip: Campo Linked Data. Indicare la lingua o le lingue principali del contenuto. Avviare la ricerca Linked Data inserendo almeno tre lettere utilizzando il codice lingua ISO (ad es. 'kmr' per il curdo settentrionale) o il nome lingua ('ital' per 'italiano' o 'lingua dei segni italiana'). Registrare tutte le lingue del contenuto, o registrare almeno la lingua principale e indicare che la risorsa è multilingue nel campo successivo. Elemento ripetibile.

Campo Linked Data: Per i termini ammissibili, aggiornati al , clicca sul seguente link per eseguire la query in tempo reale: Esegui Query (attenzione: per query complesse occorre attendere diversi secondi prima della restituzione dei risultati).

SELECT DISTINCT ?item ?itemLabel ?itemDescription ?value ?lang
WHERE {
  {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "it") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
UNION
 {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "en") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  UNION
  {
    ?item wdt:P220 ?value .
  FILTER (REGEX(?value, "***PLACEHOLDER***", "i"))  
    }  
    SERVICE wikibase:label { bd:serviceParam wikibase:language "it, en". }
BIND(LANG(?itemLabel) AS ?lang)
 }

1.7 Altro titolo

Datatype: Container

CA element code: other_title_work_CN

Ripetibilità: Sì

Quicktip: Sezione dedicata all’inserimento di uno o più titoli alternativi dell’opera. Questi possono includere: titoli paralleli (in altra lingua) varianti formali titoli abbreviati o espansi titoli storici o editoriali alternativi La presenza di più titoli va gestita con attenzione per garantire l’univocità e la corretta collocazione dell’opera nel contesto catalografico e informativo.
