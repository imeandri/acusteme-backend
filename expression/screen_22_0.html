1.5.4 suffisso

Datatype: List

CA element code: suffix_ME

Quicktip: Campo opzionale per l’aggiunta di qualificatori o suffissi che specificano ulteriormente le modalità esecutive del medium of performance. I suffissi servono a raffinare la descrizione dell’organico o dell’esecuzione, indicando caratteristiche particolari della performance come: Esecuzioni a una sola mano (one hand) Esecuzioni condivise da più musicisti su un unico strumento (two players on one instrument) Composizioni per più mani (three hands, four hands, six hands, eight hands) Strumento preparato (prepared) Registrato (recorded) Amplificato (amplified) Questi suffissi sono utilizzati per arricchire semanticamente il campo “medium of performance”, senza alterarne la struttura principale, ma aggiungendo informazioni esecutive specifiche, fondamentali per l’indicizzazione e la catalogazione analitica della musica contemporanea o sperimentale.

1.9 Rappresentazione o registrazione

CA element code: representation_recording_CN

Quicktip: Sezione strutturata dedicata alla descrizione di rappresentazioni, esecuzioni pubbliche o registrazioni audio/video legate all’opera o alla risorsa documentata. Include dati su genere di rappresentazione, data, periodo, località, sede, occasione e altre informazioni rilevanti. Se pertinente, incrociare con altri campi come Data di rappresentazione, Luogo, Sede, Occasione, Tipo di evento.

Esempi d'uso: {to be completed}

1.9.1 Tipo di rappresentazione

Datatype: List

CA element code: genre_of_representation_ME

Quicktip: Indicare il genere di rappresentazione della risorsa, scegliendo tra i valori predefiniti. Questo campo consente di distinguere la natura dell’evento o del documento sonoro/visivo associato all’opera. I valori disponibili sono: Rappresentazione: esecuzione scenica o performativa generica (teatro, musica, danza, ecc.); Prima rappresentazione: prima esecuzione pubblica documentata di un’opera; Registrazione sonora musicale: registrazione audio in studio o in contesto controllato; Registrazione dal vivo: registrazione effettuata durante un’esibizione pubblica; Rimasterizzazione: rielaborazione tecnica di una registrazione preesistente (es. miglioramento audio); Non specificato: da usare solo quando non è possibile determinare il genere di rappresentazione.

Esempi d'uso: {to be completed}

1.9.2 Anno

Il sistema supporta date-range e forme incerte (es. 1821?, 085.-089.) secondo le regole SBN e ISBD

1.10 Lingua (traduzioni)

Datatype: InformationService

CA element code: cont_lang_ME

Ripetibilità: Sì

Quicktip: Campo Linked Data. Indicare la lingua o le lingue principali del contenuto. Avviare la ricerca Linked Data inserendo almeno tre lettere utilizzando il codice lingua ISO (ad es. 'kmr' per il curdo settentrionale) o il nome lingua ('ital' per 'italiano' o 'lingua dei segni italiana'). Registrare tutte le lingue del contenuto, o registrare almeno la lingua principale e indicare che la risorsa è multilingue nel campo successivo. Elemento ripetibile.

Campo Linked Data: Per i termini ammissibili, aggiornati al , clicca sul seguente link per eseguire la query in tempo reale: Esegui Query (attenzione: per query complesse occorre attendere diversi secondi prima della restituzione dei risultati).

SELECT DISTINCT ?item ?itemLabel ?itemDescription ?value ?lang
WHERE {
  {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "it") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
UNION
 {
   ?item wdt:P220 ?value;
     rdfs:label ?itemLabel.
 FILTER((LANG(?itemLabel)) = "en") 
  FILTER(REGEX(?itemLabel, "^(***PLACEHOLDER***|[^ ]+ ***PLACEHOLDER***)", "i"))
  }
  UNION
  {
    ?item wdt:P220 ?value .
  FILTER (REGEX(?value, "***PLACEHOLDER***", "i"))  
    }  
    SERVICE wikibase:label { bd:serviceParam wikibase:language "it, en". }
BIND(LANG(?itemLabel) AS ?lang)
 }
